\chapter{Methodology}

\section{Data Source}

The empirical analysis uses the 2013 UK accident dataset. TODO: add citation for the UK accident dataset source. The dataset contains accident-level information, including accident severity and road-related variables. It is accompanied by variable documentation describing the accident-level variables and their coding.

The dataset is used as an applied setting for studying Bayesian logistic regression and Consensus Monte Carlo. The response of interest is serious/fatal accident severity relative to slight accident severity. The methodological comparison is between full-data Metropolis--Hastings sampling and Consensus Monte Carlo based on shard-level Metropolis--Hastings sampling.

\section{Data Preprocessing}

The empirical analysis uses accident severity, road type, and speed limit as the main variables. The corresponding raw variable names are \texttt{accident\_severity}, \texttt{road\_type}, and \texttt{speed\_limit}. Observations with missing values in these selected variables are removed by complete-case filtering.

Observations with unknown road type are removed because ``unknown'' is not an interpretable road-type category and cannot provide a meaningful comparison with valid road types in the logistic regression model. The number of removed observations is 226, which is small compared with the original dataset size of 82,570 observations. Therefore, removing this category is expected to have minimal effect on the overall analysis while improving interpretability and slightly reducing computational burden.

The original accident severity variable is recoded into a binary response. Serious and fatal accidents are combined into the serious/fatal category, while slight accidents are retained as the reference response class. Let $y_i$ denote the binary response for observation $i$. The response is defined as
\[
y_i =
\begin{cases}
1, & \text{if observation } i \text{ is serious or fatal},\\
0, & \text{if observation } i \text{ is slight}.
\end{cases}
\]
Thus, the event being modelled is serious/fatal accident severity.

The binary response is imbalanced. The serious/fatal category contains 10,970 observations, corresponding to 0.1332 of the analysis data, while the slight category contains 71,374 observations, corresponding to 0.8668. Thus, serious/fatal accidents form the minority class and slight accidents form the majority class. However, the serious/fatal category still contains a sufficiently large number of observations for posterior simulation in this study, so no special remedy such as resampling or class weighting is applied.

Road type and speed limit are treated as categorical predictors. The reference category for road type is single carriageway, and the reference category for speed limit is category 2. A design matrix $\mathbf{X}$ is constructed from the fitted logistic regression model and includes the intercept together with indicator variables for the non-reference predictor categories.

\section{Bayesian Logistic Regression Model}

Let $\mathbf{y}=(y_1,\ldots,y_n)^T$ denote the binary response vector, and let $\mathbf{x}_i$ denote the predictor vector for observation $i$, including the intercept term and the encoded categorical predictors. Let $\boldsymbol{\beta}=(\beta_0,\beta_1,\ldots,\beta_p)^T$ denote the regression coefficient vector, where $d=p+1$ is the number of regression coefficients. The Bayesian logistic regression model assumes
\[
y_i \mid \boldsymbol{\beta} \sim \operatorname{Bernoulli}(p_i),
\]
where $p_i$ is the probability that observation $i$ has serious/fatal accident severity. The logistic link is
\[
\log\left(\frac{p_i}{1-p_i}\right)
=
\beta_0+\beta_1x_{i1}+\beta_2x_{i2}+\cdots+\beta_px_{ip}
=
\mathbf{x}_i^T\boldsymbol{\beta}.
\]
There is no additive error term in this link equation because the stochastic component is represented through the Bernoulli response distribution.

The event probability can be written in sigmoid form as
\[
p_i
=
\frac{\exp(\mathbf{x}_i^T\boldsymbol{\beta})}
{1+\exp(\mathbf{x}_i^T\boldsymbol{\beta})}
=
\frac{1}{1+\exp(-\mathbf{x}_i^T\boldsymbol{\beta})}.
\]

For $n$ independent observations, the likelihood is
\[
p(\mathbf{y}\mid \boldsymbol{\beta})
=
\prod_{i=1}^{n}
p_i^{y_i}(1-p_i)^{1-y_i}.
\]
The corresponding log-likelihood is
\[
\log p(\mathbf{y}\mid \boldsymbol{\beta})
=
\sum_{i=1}^{n}
\left[
y_i\mathbf{x}_i^T\boldsymbol{\beta}
-
\log\{1+\exp(\mathbf{x}_i^T\boldsymbol{\beta})\}
\right].
\]
In the computation, the term $\log\{1+\exp(\mathbf{x}_i^T\boldsymbol{\beta})\}$ is evaluated using a numerically stable form. TODO: add citation for Bayesian logistic regression.

\section{Prior Distribution}

A weakly informative normal prior distribution is assigned independently to each regression coefficient. For $j=0,\ldots,p$,
\[
\beta_j \sim N(0,2.5^2).
\]
This prior is centred at zero and provides mild regularisation for the logistic regression coefficients. The prior scale is motivated by weakly informative prior ideas for logistic regression, although the current implementation uses a normal prior with standard deviation 2.5 rather than the Cauchy prior discussed by Gelman et al. \cite{gelman2008weakly}.

Under prior independence, the joint prior density is
\[
\pi(\boldsymbol{\beta})
=
\prod_{j=0}^{p}
\pi(\beta_j).
\]

\section{Posterior Distribution}

Treating the predictor values as fixed and incorporated into the likelihood, the posterior distribution for the regression coefficient vector is
\[
\pi(\boldsymbol{\beta}\mid \mathbf{y})
\propto
p(\mathbf{y}\mid \boldsymbol{\beta})\pi(\boldsymbol{\beta}).
\]
Analytical posterior summaries are not used in this study. Instead, posterior summaries are approximated from samples generated by Metropolis--Hastings algorithms. TODO: add citation for posterior simulation.

\section{Full-Data Metropolis--Hastings Sampling}

The full-data analysis uses the complete response vector $\mathbf{y}$ to sample from $\pi(\boldsymbol{\beta}\mid \mathbf{y})$, with the fixed predictor information incorporated into the likelihood. Two full-data samplers are implemented: Random Walk Metropolis--Hastings and Independent Metropolis--Hastings. TODO: add citation for the Metropolis--Hastings algorithm.

In the following sections,
\[
\operatorname{MVN}(\boldsymbol{\mu},\boldsymbol{\Sigma})
\]
denotes a multivariate normal distribution with mean vector $\boldsymbol{\mu}$ and covariance matrix $\boldsymbol{\Sigma}$. A frequentist logistic regression model is fitted before posterior simulation. Let $\widehat{\boldsymbol{\beta}}$ denote the estimated coefficient vector from this model, and let $\widehat{\boldsymbol{\Sigma}}$ denote the estimated covariance matrix of $\widehat{\boldsymbol{\beta}}$. This covariance matrix is used as a reference covariance matrix for constructing Metropolis--Hastings proposals because it provides an empirical estimate of the posterior scale and correlation structure. If any starting value is non-finite, it is replaced by zero before sampling.

For both full-data samplers, the number of Metropolis--Hastings iterations is set to 100000, and the burn-in fraction is 0.2. Runtime is recorded using start and end time measurements, and the recorded runtime values are reported in Chapter 4 as part of the computational comparison.

\subsection{Random Walk Metropolis--Hastings Algorithm}

In Random Walk Metropolis--Hastings, the proposed value at iteration $t$ is generated by perturbing the current state:
\[
\boldsymbol{\beta}^{*}
=
\boldsymbol{\beta}^{(t-1)}+\boldsymbol{\varepsilon},
\qquad
\boldsymbol{\varepsilon}
\sim
\operatorname{MVN}(\mathbf{0},\boldsymbol{\Sigma}_{\mathrm{RWMH}}).
\]
The full-data Random Walk Metropolis--Hastings proposal covariance matrix is
\[
\boldsymbol{\Sigma}_{\mathrm{RWMH}}
=
c_{\mathrm{RWMH}}^2
\frac{2.38^2}{d}
\widehat{\boldsymbol{\Sigma}}.
\]
In this study, $c_{\mathrm{RWMH}}=0.75$. This tuning factor adjusts the proposal scale so that the random walk proposal is not too wide for the posterior target. Reducing the proposal scale can improve acceptance behaviour when the initial proposal covariance is too large, but no claim of optimality is made.

Because the random walk proposal is symmetric, the proposal densities cancel in the Metropolis--Hastings ratio. The acceptance probability is
\[
\alpha
=
\min\left\{
1,
\frac{
\pi(\boldsymbol{\beta}^{*}\mid \mathbf{y})
}{
\pi(\boldsymbol{\beta}^{(t-1)}\mid \mathbf{y})
}
\right\}.
\]
In log form, the proposal is accepted when
\[
\log u
<
\log \pi(\boldsymbol{\beta}^{*}\mid \mathbf{y})
-
\log \pi(\boldsymbol{\beta}^{(t-1)}\mid \mathbf{y}),
\]
where $u \sim \operatorname{Uniform}(0,1)$.

\subsection{Independent Metropolis--Hastings Algorithm}

In Independent Metropolis--Hastings, the proposal distribution does not depend on the current state. The full-data Independent Metropolis--Hastings sampler uses
\[
\boldsymbol{\beta}^{*}
\sim
\operatorname{MVN}(
\widehat{\boldsymbol{\beta}},
\boldsymbol{\Sigma}_{\mathrm{IMH}}
),
\]
where
\[
\boldsymbol{\Sigma}_{\mathrm{IMH}}
=
c_{\mathrm{IMH}}^2
\widehat{\boldsymbol{\Sigma}}.
\]
In this study, $c_{\mathrm{IMH}}=1.5$. A wider variance-covariance matrix is often useful for an independent proposal because it helps the proposal explore the posterior support rather than being too concentrated near the frequentist estimate.

Since the proposal is independent rather than a symmetric random walk, the proposal-density correction is included in the Metropolis--Hastings acceptance probability. If $q(\cdot)$ denotes the independent proposal density, then
\[
\alpha
=
\min\left\{
1,
\frac{
\pi(\boldsymbol{\beta}^{*}\mid \mathbf{y})q(\boldsymbol{\beta}^{(t-1)})
}{
\pi(\boldsymbol{\beta}^{(t-1)}\mid \mathbf{y})q(\boldsymbol{\beta}^{*})
}
\right\}.
\]
Equivalently, the log acceptance ratio is
\[
\log \pi(\boldsymbol{\beta}^{*}\mid \mathbf{y})
-
\log \pi(\boldsymbol{\beta}^{(t-1)}\mid \mathbf{y})
+
\log q(\boldsymbol{\beta}^{(t-1)})
-
\log q(\boldsymbol{\beta}^{*}).
\]

\section{Consensus Monte Carlo}

Consensus Monte Carlo is used to approximate the full posterior distribution by partitioning the data, sampling from shard-specific subposteriors, and combining the resulting subposterior samples \cite{scott2017comparing,scott2022bayes}. The data are divided into $K$ subsets, commonly referred to as shards. In this study, Consensus Monte Carlo is implemented with both Random Walk Metropolis--Hastings and Independent Metropolis--Hastings shard-level samplers.

\subsection{Data Partitioning}

Let the full dataset be partitioned into $K$ shards. In the empirical analysis, $K=4$. The response vector for shard $k$ is denoted by $\mathbf{y}_k$, where $k=1,\ldots,K$. Observations are randomly assigned to shards before the shard-level samplers are run.

The full likelihood factorises across the shards as
\[
p(\mathbf{y}\mid \boldsymbol{\beta})
=
\prod_{k=1}^{K}
p(\mathbf{y}_k\mid \boldsymbol{\beta}),
\]
where each shard likelihood is computed using the observations assigned to that shard.

\subsection{Subposterior Distribution}

For shard $k$, the subposterior distribution is defined by combining the shard likelihood with a fractional prior contribution:
\[
\pi_k(\boldsymbol{\beta}\mid \mathbf{y}_k)
\propto
p(\mathbf{y}_k\mid \boldsymbol{\beta})
\pi(\boldsymbol{\beta})^{1/K}.
\]
The factor $\pi(\boldsymbol{\beta})^{1/K}$ splits the prior contribution across shards. Under the likelihood factorisation,
\[
\prod_{k=1}^{K}
\pi_k(\boldsymbol{\beta}\mid \mathbf{y}_k)
\propto
\prod_{k=1}^{K}
p(\mathbf{y}_k\mid \boldsymbol{\beta})
\pi(\boldsymbol{\beta})^{1/K}
=
p(\mathbf{y}\mid \boldsymbol{\beta})\pi(\boldsymbol{\beta})
\propto
\pi(\boldsymbol{\beta}\mid \mathbf{y}).
\]
Thus, multiplying the subposterior kernels recovers a kernel proportional to the full-data posterior.

\subsection{Shard-level Metropolis--Hastings Sampling}

Separate Metropolis--Hastings chains are run on the shard-level subposterior distributions. For each shard-level sampler, the target density in the acceptance probability is $\pi_k(\boldsymbol{\beta}\mid \mathbf{y}_k)$ rather than the full-data posterior density.

For Consensus Monte Carlo using Random Walk Metropolis--Hastings, the proposal covariance matrix is based on the full-data covariance estimate and is scaled by $K$:
\[
\boldsymbol{\Sigma}_{\mathrm{CMC,RWMH}}
=
c_{\mathrm{CMC,RWMH}}^2
\frac{2.38^2}{d}
K\widehat{\boldsymbol{\Sigma}}.
\]
In this study, $c_{\mathrm{CMC,RWMH}}=0.75$. The scaling by $K$ is motivated by the fact that each shard contains approximately $n/K$ observations. Posterior uncertainty is inversely related to information size, so when the sample size decreases from $n$ to approximately $n/K$, the variance scale is expected to increase by approximately $K$. This is analogous to sample variance or covariance scaling, where fewer observations give a larger uncertainty scale. Since each shard is part of the full dataset, the full-data covariance structure remains a useful reference for correlations among coefficients. This reasoning motivates the proposal scale but does not make it exact.

For Consensus Monte Carlo using Independent Metropolis--Hastings, a separate frequentist logistic regression model is fitted within each shard. This gives a shard-specific coefficient estimate $\widehat{\boldsymbol{\beta}}_k$ and covariance estimate $\widehat{\boldsymbol{\Sigma}}_k$. It is reasonable to use these quantities for the independent proposal because they are estimated directly from the same shard that defines the target subposterior. The proposal is
\[
\boldsymbol{\beta}^{*}
\sim
\operatorname{MVN}(
\widehat{\boldsymbol{\beta}}_k,
\boldsymbol{\Sigma}_{\mathrm{CMC,IMH},k}
),
\]
where
\[
\boldsymbol{\Sigma}_{\mathrm{CMC,IMH},k}
=
c_{\mathrm{CMC,IMH}}^2
\widehat{\boldsymbol{\Sigma}}_k.
\]
In the current implementation, $c_{\mathrm{CMC,IMH}}=1.5$. The same principle as the full-data Independent Metropolis--Hastings proposal is used: the proposal is centred at a logistic-regression estimate and uses a covariance estimate from that fitted model, but both quantities are shard-specific. Since the proposal is independent of the current state, the proposal-density correction is included in the Metropolis--Hastings acceptance probability.

The shard-level samplers use the same number of iterations and burn-in fraction as the full-data samplers. After burn-in is removed, the retained subposterior samples are aligned by using the common retained sample length across shards.

\subsection{Consensus Posterior Combination}

The retained subposterior samples are combined using a precision-weighted consensus method. Let $S$ denote the common number of retained samples, and let $\boldsymbol{\beta}_k^{(s)}$ denote the $s$th retained subposterior sample from shard $k$, where $s=1,\ldots,S$. Let
\[
\widehat{\boldsymbol{\Sigma}}_k^{\mathrm{sub}}
\]
denote the estimated covariance matrix of the retained subposterior samples from shard $k$. The corresponding precision matrix is
\[
\mathbf{W}_k
=
\left(
\widehat{\boldsymbol{\Sigma}}_k^{\mathrm{sub}}
\right)^{-1}.
\]
The precision-weighted consensus sample is
\[
\boldsymbol{\beta}_{\mathrm{consensus}}^{(s)}
=
\left(
\sum_{k=1}^{K}
\mathbf{W}_k
\right)^{-1}
\sum_{k=1}^{K}
\mathbf{W}_k
\boldsymbol{\beta}_k^{(s)}.
\]
This produces the consensus posterior sample sequence
\[
\left\{
\boldsymbol{\beta}_{\mathrm{consensus}}^{(s)}
\right\}_{s=1}^{S},
\]
which is summarised and compared with the full-data posterior samples.

\section{Posterior Summary and Evaluation Criteria}

Posterior samples from the full-data Metropolis--Hastings samplers and the Consensus Monte Carlo procedures are evaluated using posterior summaries and diagnostic quantities. For each regression coefficient, the summaries are:
\begin{enumerate}
\item posterior mean;
\item posterior median;
\item posterior standard deviation;
\item effective sample size;
\item Monte Carlo standard error;
\item 95\% credible interval;
\item odds ratio;
\item 95\% odds-ratio interval.
\end{enumerate}

The effective sample size is estimated from the autocorrelation structure of the sampled values. The Monte Carlo standard error is computed as the posterior standard deviation divided by the square root of the estimated effective sample size. Odds ratios are obtained by exponentiating posterior means, and the 95\% odds-ratio intervals are obtained by exponentiating the endpoints of the 95\% credible intervals.

Additional evaluation criteria include acceptance rate, runtime, trace plots, posterior histograms, and posterior density plots. Acceptance rates are used to assess the behaviour of the Metropolis--Hastings proposal mechanisms. Runtime is recorded during the empirical analysis and reported in Chapter 4. Histogram and trace plots are produced for the full-data Metropolis--Hastings chains, while posterior density plots are produced for the combined consensus samples. Since the Consensus Monte Carlo implementation is sequential, the runtime comparison is not interpreted as evidence of parallel speedup.
