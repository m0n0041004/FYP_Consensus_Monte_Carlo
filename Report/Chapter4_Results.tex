\section{Data Summary}

This chapter reports the empirical results for Bayesian logistic regression applied to the 2013 UK accident dataset. The response is binary accident severity, where serious/fatal accident severity is coded as 1 and is the event being modelled, while slight accident severity is coded as 0 and is the reference response class. The predictors are road type and speed limit.

The exported model settings report 82,570 original observations and 82,344 observations after preprocessing. Serious/fatal accidents account for 10,970 observations, corresponding to 13.32\% of the analysis data, while slight accidents account for 71,374 observations, corresponding to 86.68\%. The shard acceptance rates for CMC-RWMH are 0.3895, 0.3911, 0.4007, and 0.3960. The shard acceptance rates for CMC-IMH are 0.2178, 0.2154, 0.2170, and 0.2188.

The implementation is sequential because no multi-machine computing environment is used. It is used to mimic the data-partitioning and subposterior-combination structure of Consensus Monte Carlo, not to measure actual parallel execution. Since CMC uses $K=4$ shards, Table \ref{tab:acceptance_runtime} also reports an approximate per-shard runtime, computed as total sequential CMC runtime divided by $K$. This approximation is used only to illustrate the potential computational advantage of distributed execution and does not represent an actual measured parallel runtime.

The total sequential CMC runtimes are similar to the full-data runtimes in this implementation. In an ideal parallel implementation, shard-level chains could run simultaneously, so wall-clock time may be closer to the maximum shard runtime rather than the sum of shard runtimes. However, actual speedup depends on hardware, communication cost, posterior combination cost, implementation details, and statistical efficiency. Therefore, the reported timings do not demonstrate true parallel speedup and should not be interpreted as implying automatic $K$-fold acceleration.

\section{Summary of Findings}

The frequentist logistic regression provides an initial baseline, diagnostic checks, and proposal information for the Metropolis--Hastings samplers. Its p-values are reported descriptively, while the likelihood-ratio G-square test assesses improvement over the intercept-only model and the GVIF values assess multicollinearity among the categorical predictors.

The full-data Random Walk Metropolis--Hastings and full-data Independent Metropolis--Hastings samplers produce similar posterior means, medians, standard deviations, and credible intervals. Across the Bayesian posterior summaries, the 95\% credible interval for Road type: one way street contains zero, while the intervals for the other non-intercept coefficients exclude zero. This provides posterior evidence that those coefficients are credibly different from zero under the fitted model.

The Consensus Monte Carlo summaries are compared against the full-data methods as posterior approximations. CMC-RWMH and CMC-IMH are generally close to the full-data posterior summaries, with the more visible mean differences appearing for Road type: slip road, Speed limit: 1, and Speed limit: 6. The acceptance rates for the CMC methods are similar to the corresponding full-data Metropolis--Hastings methods and should still be interpreted together with posterior summaries, density plots, and runtime.

The runtime results reflect a sequential implementation and do not prove true parallel speedup. The approximate per-shard runtime is included only as an indicator of possible distributed computational advantage under a parallel implementation. Actual speedup would depend on the computing environment, communication overhead, posterior combination cost, implementation details, and statistical efficiency.
